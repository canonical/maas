Delete a tag

```bash
maas $PROFILE tag delete [--help] [-d] [-k] name

```

#### **Positional arguments**
| Argument | Effect |
|---|---|
| name | The name of the resource (e.g., `my-machine`, `my-zone`) |


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Deletes a tag by name.

Update a tag

```bash
maas $PROFILE tag update [--help] [-d] [-k] name [data ...]

```

#### **Positional arguments**
| Argument | Effect |
|---|---|
| name | The name of the resource (e.g., `my-machine`, `my-zone`) |


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Update elements of a given tag.

##### Keyword "name"
Optional String. The new tag name. Because<br>the name will be used in urls, it should be short.
##### Keyword "comment"
Optional String. A description of what the tag will be used for in natural language.
##### Keyword "definition"
Optional String. An XPATH query that is<br>evaluated against the hardware_details stored for all nodes<br>(i.e. the output of ``lshw -xml``).


Note: This command accepts JSON.


Update nodes associated with this tag

```bash
maas $PROFILE tag update-nodes [--help] [-d] [-k] name [data ...]

```

#### **Positional arguments**
| Argument | Effect |
|---|---|
| name | The name of the resource (e.g., `my-machine`, `my-zone`) |


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Add or remove nodes associated with the given tag.<br>Note that you must supply either the ``add`` or ``remove``<br>parameter.

##### Keyword "add"
Optional String. The system_id to tag.
##### Keyword "remove"
Optional String. The system_id to untag.
##### Keyword "definition"
Optional String. If given, the<br>definition (XPATH expression) will be validated against the<br>current definition of the tag. If the value does not match, MAAS<br>assumes the worker is out of date and will drop the update.
##### Keyword "rack_controller"
Optional String. The system ID<br>of the rack controller that processed the given tag initially.<br>If not given, the requester must be a MAAS admin. If given,<br>the requester must be the rack controller.


Note: This command accepts JSON.


Create a new tag

```bash
maas $PROFILE tags create [--help] [-d] [-k] [data ...]

```


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Create a new tag.

##### Keyword "name"
Optional String. The new tag name. Because<br>the name will be used in urls, it should be short.
##### Keyword "comment"
Optional String. A description of what the tag will be used for in natural language.
##### Keyword "definition"
Optional String. An XPATH query that is<br>evaluated against the hardware_details stored for all nodes<br>(i.e. the output of ``lshw -xml``).
##### Keyword "kernel_opts"
Optional String. Nodes associated<br>with this tag will add this string to their kernel options<br>when booting. The value overrides the global ``kernel_opts``<br>setting. If more than one tag is associated with a node, command<br>line will be concatenated from all associated tags, in alphabetic<br>tag name order.


Note: This command accepts JSON.
