Delete a user

```bash
maas $PROFILE user delete [--help] [-d] [-k] username [data ...]

```

#### **Positional arguments**
| Argument | Effect |
|---|---|
| username | The username parameter |


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Deletes a given username.

##### Keyword "transfer_resources_to"
Optional String. An optional<br>username. If supplied, the allocated resources of the user being<br>deleted will be transferred to this user. A user can't be removed<br>unless its resources (machines, IP addresses, â€¦), are released or<br>transferred to another user.


Note: This command accepts JSON.

Create a MAAS user account

```bash
maas $PROFILE users create [--help] [-d] [-k] [data ...]

```


#### **Command-line options**
| Option | Effect |
|---|---|
| --help, -h | Show this help message and exit. |
| -d, --debug | Display more information about API responses. |
| -k, --insecure | Disable SSL certificate check. |


#### **Keywords**
Creates a MAAS user account. This is not safe: the password is sent in plaintext. Avoid it for<br>production, unless you are confident that you can prevent eavesdroppers<br>from observing the request.

##### Keyword "username"
Optional String. Identifier-style username<br>for the new user.
##### Keyword "email"
Optional String. Email address for the new user.
##### Keyword "password"
Optional String. Password for the new user.
##### Keyword "is_superuser"
Optional Boolean. Whether the new user is<br>to be an administrator. ('0' == False, '1' == True)


Note: This command accepts JSON.
